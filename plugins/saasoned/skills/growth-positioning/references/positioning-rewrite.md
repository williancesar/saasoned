# Positioning-rewrite procedure (worked)

A deterministic procedure that takes a vague value proposition and emits two
artifacts: a **5(+1) positioning table** (Dunford) and a **Moore elevator
pitch**. Use this whenever the request is "make this clearer / fix our
messaging / who is this for."

## Inputs required

- The current (vague) description.
- Who currently gets the most value ("the lovers"). If unknown → stop and route
  to `growth-customer-discovery`; you cannot fabricate component 4.

## Steps

1. **Lovers → evidence.** Note 1–3 real users/segments who get outsized value
   and why (specifics, not guesses).
2. **True competitive alternatives.** What would they *actually* do without you?
   Force past "rival apps" to the real default (spreadsheet, group chat, manual
   habit, nothing).
3. **Unique attributes.** What you have that those alternatives lack. Concrete.
4. **Attribute → value → proof.** One row each. Drop attributes with no value or
   no proof.
5. **Best-fit characteristics.** The observable traits predicting who cares most
   (so a channel can target them — feeds `growth-traction-channels`).
6. **Market category.** The frame that makes the value obvious to the best-fit
   customer. Prefer **big-fish-small-pond** unless you can lead a category or
   fund category creation.
7. **Trend (optional).** Only a real wave the customer already cares about.
8. **Beachhead.** Narrow the best-fit to the single segment you can dominate
   first (Moore). Smaller-but-dominable wins.
9. **Whole product.** What that beachhead needs beyond the core to fully adopt.
10. **Emit artifacts** (table + pitch below).

## Output template

**5(+1) table**

| Component | Value |
|---|---|
| Competitive alternatives | … |
| Unique attributes | … |
| Value & proof | … |
| Best-fit customer | … |
| Market category | … |
| Trend (optional) | … / none |
| Beachhead (Moore) | … |
| Whole product gap | … |

**Moore elevator pitch**

> For [beachhead] who [need], [product] is a [category] that [key benefit].
> Unlike [real alternative], our product [primary differentiation].

## Worked example (illustrative — not a product spec)

Input: "*vai-nascer-v2* is a pregnancy app with tracking and family features."

| Component | Value |
|---|---|
| Competitive alternatives | Generic week-by-week trackers; family group chat; nothing shared |
| Unique attributes | Family-member access tied to the pregnant woman's single subscription/trial |
| Value & proof | Partner/family stay synced with one purchase decision; proof = family-member activation rate per subscriber in a pilot |
| Best-fit customer | First-trimester pregnant women in Brazil who want family involved |
| Market category | Shared pregnancy companion (big-fish-small-pond, not "pregnancy app") |
| Trend | None forced |
| Beachhead | Exactly that first-trimester, family-oriented segment |
| Whole product gap | Easy family-invite flow; partner onboarding; trust/privacy assurances |

> For first-trimester pregnant women in Brazil who want their partner and family
> involved without everyone managing a separate app, vai-nascer is a shared
> pregnancy companion that keeps the whole family synced on one subscription.
> Unlike generic trackers and group chats, our product ties family access to the
> mother's plan, so involvement is one decision, not many.

## Common failure it prevents

Rewriting copy (surface) when the real defect is an unanswered component
(usually category or best-fit) — producing prettier words that still don't land.
