# The Five-Stage Arc — Chen

**Source.** Andrew Chen, *The Cold Start Problem*. A network product moves
through five stages; the bottleneck and the correct tactics differ at each, so
**locating the stage is prerequisite to any advice**.

```
Cold Start → Tipping Point → Escape Velocity → Hitting the Ceiling → The Moat
```

---

## 1. Cold Start

**Definition.** Below critical mass: too few of the right users for the network
to deliver value; it decays without constant pushing.
**Bottleneck.** Creating the first stable atomic network.
**Tactics.** Seed one bounded network; hard side first; unscalable acts;
constrain scope. (See atomic-network.md.)
**Decision cue.** No atomic network is self-sustaining → you are here → only
valid topic is seeding; refuse scale talk (Guardrail 2).
**Failure it prevents.** Spending on growth before any network holds.

## 2. Tipping Point

**Definition.** A repeatable way to spin up new atomic networks reliably; each
new one reaches density without heroics.
**Bottleneck.** Making network creation repeatable, not one-off.
**Tactics.** Find the replicable seeding playbook; attack adjacent networks;
invest where networks tip themselves.
**Decision cue.** One network works but each new one needs manual heroics → not
yet tipped; systematize before scaling spend.
**Failure it prevents.** Mistaking one hand-built network for a scalable model.

## 3. Escape Velocity

**Definition.** Network effects compound; growth and engagement reinforce
themselves. Work shifts to *strengthening* the effects (acquisition, engagement,
economic loops), not proving them.
**Bottleneck.** Maximizing the strength of the loops you now have.
**Tactics.** Improve each network-effect loop deliberately; deepen the hard
side's reasons to stay.
**Decision cue.** Self-reinforcing growth present → optimize loops; this is
where scaling spend is finally justified.
**Failure it prevents.** Coasting and under-investing while the flywheel is
actually fragile.

## 4. Hitting the Ceiling

**Definition.** Growth saturates or degrades — market saturation, channel
decay, congestion, or **anti-network effects** (see
network-effect-taxonomy.md).
**Bottleneck.** Diagnosing *which* ceiling, not adding more users.
**Tactics.** Fix quality/congestion; new networks/segments; rework saturated
channels.
**Decision cue.** "More users but flat or worse value" → ceiling, not seeding;
do **not** prescribe seeding tactics here.
**Failure it prevents.** Treating a saturation/quality ceiling as a cold-start
problem and pouring in more users that make it worse.

## 5. The Moat

**Definition.** Mature network effects become a durable competitive defense;
the work is leveraging and protecting them against competitors and decay.
**Bottleneck.** Defending density and switching costs.
**Tactics.** Reinforce hard-side lock-in, raise the cost of multi-homing,
defend the most valuable atomic networks.
**Decision cue.** Defensibility, not growth, is the question → competitive
defense framing.
**Failure it prevents.** Assuming network effects are permanent and neglecting
defense until a competitor peels off the hard side.

---

## Using the arc

State the stage explicitly before giving tactics. The most common error is
applying **Cold Start seeding advice to a Ceiling-stage problem** (or scaling
advice to a Cold Start). Wrong stage → wrong tactic, regardless of how good the
tactic is for its own stage.
