# Fogg Behavior Model (B = MAT)

**Source.** B.J. Fogg, as applied in Eyal, *Hooked* (the Action stage).

**Definition.** A behavior occurs only when three factors are present *at the
same moment*: sufficient **Motivation**, sufficient **Ability** (the behavior is
easy enough), and a **Trigger** to act. Written **B = M·A·T** — if any factor is
too low, the behavior doesn't happen even if the others are high.

- **Motivation** axes: seek pleasure/avoid pain; seek hope/avoid fear; seek
  social acceptance/avoid rejection.
- **Ability** levers (reduce the cost of acting): time, money, physical effort,
  brain cycles, social deviance, non-routineness. Simplicity = the scarcest of
  these for that user, removed.
- **Trigger** types: spark (with motivation-boost), facilitator (with
  ability-boost), signal (only a reminder, for already-able-and-motivated).

**When it applies.** Any "users don't perform the key action" diagnosis; CTA/
onboarding step design.

**When it does NOT.** When the user fundamentally doesn't want the outcome — no
trigger or simplification fixes a value problem; route to
`growth-customer-discovery`.

**Decision cue (default stance).** Diagnose the missing factor, then:

```
Action not happening?
├─ No trigger at the right moment        → add a well-timed trigger
├─ Motivated but it's too hard (Ability) → SIMPLIFY first (default lever)
└─ Able but doesn't care (Motivation)    → check it's a real want
                                            (else → customer-discovery)
```

Default to **increasing ability (reducing friction)** before adding motivation:
simplicity scales and doesn't fatigue; motivation is expensive and decays.

**Worked micro-example.** *family-agent*: a parent doesn't add the second
guardian. Motivation is high (they want shared coordination); the blocker is
**ability** (multi-step invite). Fix = one-tap invite link, not a persuasive
email. Adding motivation here would have been the inverted error.

**Common failure it prevents.** Throwing motivation (incentives, persuasive
copy, more notifications) at what is actually a friction/ability problem —
expensive and ineffective.
