# The Manipulation Matrix — the ethics gate

**Source.** Nir Eyal, *Hooked*. This is a **mandatory gate**: never recommend a
habit loop without running it through this matrix and stating the quadrant.

**Definition.** Two yes/no questions about the maker building the habit:

- **Would the maker use the product themselves?** (Do *I* use it?)
- **Does it materially improve the user's life?** (Does it make their life
  better, by the user's own standard — not the maker's revenue?)

Crossing them gives four roles:

| | Maker uses it | Maker won't use it |
|---|---|---|
| **Improves user's life** | **Facilitator** | **Peddler** |
| **Doesn't improve life** | **Entertainer** | **Dealer** |

- **Facilitator** — build it. The healthiest quadrant.
- **Peddler** — proceed only with caution: usually a signal the maker hasn't
  found the real value or audience (often a `growth-positioning` /
  `growth-customer-discovery` gap), or is rationalizing.
- **Entertainer** — permissible *only* if honestly framed as entertainment and
  the user knowingly opts into it; not to be disguised as utility.
- **Dealer** — **refuse.** Designing a habit that doesn't help the user and that
  the maker wouldn't use is manipulation. Do not produce the loop.

**When it applies.** Every time a habit/engagement loop is proposed,
recommended, or audited — without exception.

**When it does NOT.** Never skipped. (There is no "internal tool exemption":
even captive users deserve the gate.)

**Decision cue.**

```
Run BOTH questions →
  Facilitator → recommend
  Peddler     → recommend only with explicit caution + flag the likely
                value/positioning gap
  Entertainer → only if honestly labeled as entertainment, user opts in
  Dealer      → REFUSE; cite Hooked; redirect to delivering real value
```

State the quadrant explicitly in any habit-loop recommendation. If the answer to
"does it improve the user's life" depends on facts you don't have, get them
(`growth-customer-discovery`) before recommending — don't assume yes.

**Worked micro-example.** A request to make *vai-nascer-v2* send escalating
guilt-framed streak alerts to keep a postpartum mother opening the app daily
with no added value: maker wouldn't subject themselves to it; doesn't improve
her life → **Dealer** → refuse, cite *Hooked*, propose a value-first
alternative (genuinely useful, frequency-matched content) instead.

**Common failure it prevents.** Shipping a technically complete, effective Hook
that exploits the user — the exact outcome the matrix exists to stop.
